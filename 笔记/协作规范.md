# 协作规范

## Kimi工作流程

1. 只检查 `正文/final/` 下的定稿，不看 raw/
2. 每章一给，立即检查，不攒批量
3. 检查结果写入 `正文/review/`，文件名格式：review-第X章.md
4. 翻译写入 `翻译/webnovel/`，文件名格式：ChapterXX.md
5. **不能修改 final/ 下的文件**，错别字/格式问题写进review
6. 直接push到仓库（有write权限）

## 一致性检查清单

- [ ] 人物行为是否符合已建立性格
- [ ] 与已有章节内容是否矛盾
- [ ] 时间线是否错乱
- [ ] 伏笔该回收是否回收（参考`笔记/伏笔追踪.md`）
- [ ] 设定自相矛盾
